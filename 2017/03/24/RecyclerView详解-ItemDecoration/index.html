RecyclerView详解--ItemDecoration

RecyclerView(RV)相对原来的ListView提供了更好的缓存机制，将布局管理拆分到LayoutManager中使扩展更加方便，同时Google提供了针对Linear、Grid、StaggeredGrid不同布局需求的实现，使我们对于列表布局的使用更加便捷，Adapter中也提供了针对ViewHolder的抽取和封装，也提供了针对item的动画、装饰等特性。接下来的几篇文章，让我们一一对RecyclerView进行分析。

RecyclerView的类结构图：

从上面的结构图可以看出，大部分实现类都定义在RV中以内部类存在。我们平时开发比较常用的Adapter、ViewHolder、LayoutManager、ItemDecoration、ItemAnimator等，这些是我们平时使用RV时用的最多的。今天我们就从一个最简单的入手来分析RecyclerView—ItemDecoration。

ItemDecoration简介

ItemDecoration是item的装饰器，平常的开发中，我们更多是使用它去绘制一条分隔线，或者绘制item之间的间隔，当然，这些是最简单的使用方式，既然是装饰器，我们就可以在每个Item上进行不同的装饰操作，有了它我们在开发时就可以多一个角度去考虑一些实现，比如后面的示例：使用Decoration实现一个物流详情界面。

先看一下添加ItemDecoration的入口，addItemDecoration(ItemDecoration decor)具体实现：

调用该方法最终把定义的decor都放在了RV内部的集合变量mItemDecorations中，由这点也可以看出，我们并不是只能定义一个decor，我们可以添加多个不同的decor，它们最终都会装饰到RV的item上。继续跟踪decor中3个方法的调用位置。

onDraw(Canvas c, RecyclerView parent, State state)

看名称就知道这个肯定是让我们用来绘制具体的装饰的，传递了Canvas进来，我们可以绘制一切在画布上，同时也将RV自身传递进来，该方法调用在RV的onDraw()中：

onDrawOver(Canvas c, RecyclerView parent, State state)

该方法也是用来绘制的，但是它绘制的装饰物，最终是显示在ItemView之上的，这就是over的含义。该方法调用在RV的draw()方法中：

getItemOffsets(Rect outRect, View view, RecyclerView parent, State state)

该方法最终的含义是为ItemView添加额外的padding，这个padding需要我们设置到入参outRect中,我们可以跟随源码进行查看，该方法调用在RV的getItemDecorInsetsForChild()方法中：

而getItemDecorInsetsForChild()方法是在LayoutManager的measureChild()方法中调用的：

从上面的代码可以看出，在child进行测量时，会将decor中设置的值作为padding传递进去。

我们都清楚View的测量和绘制流程：measure —>layout —>draw，所以结合前面的源码，测量流程是发生在draw之前的，onDraw()会在view的draw()方法中调用。

所以ItemDecoration中3个方法的调用顺序是：getItemOffsets — > onDraw — > onDrawOver 。

具体绘制我们可以参考官方提供的DividerItemDecoration类，这里只说明一点，我们在自定义ItemDecoration时需要将getItemOffsets中设置的Rect当做为ItemView设置padding就行了 ，如果只是设置间隔，可以不用实现onDraw()，只需返回left、top、right、bottom的padding值，如果需要绘制，getItemOffsets中也必须返回绘制的区域，即padding所占的区域，否则ondraw()方法中绘制的内容无法正常显示出来。

总结

通过对ItemDecoration的分析，我们看到了RecyclerView所抽象出来的设计非常便利我们扩展，将各种功能模块分离，降低耦合，我们可以针对各个子功能进行自己的扩展实现。

而且在了解了ItemDecoration后，我们也可以看到，它并不仅仅是用来绘制分隔线，使用它可以绘制一切我们想要的装饰，给了我们一种新的布局角度。
