使用Travis CI自动构建Github项目

Travis CI是目前最流行的持续集成构建工具，它分免费版(https://travis-ci.org)和付费版(https://travis-ci.org)，免费版连接的Github，使用Github账号登录后，会自动关联Github上的所有项目，可以将这些项目纳入到持续集成构建中，之所以有收费版是因为免费版关联Github上的项目都是开源的，作为公司开发的企业项目，肯定不可能开源到Github上面。收费版肯定是解决这个问题的，它提供针对私有项目的持续集成服务，而且费用不低。对于个人开源的项目使用Travis CI是个不错的选择。

下面介绍使用Travis构建Android项目的具体步骤。

发布Github Releases

上面的两步指定了具体的配置任务，如:打一个release包，但是在travis自动执行构建后人会清理掉所有产生的文件，所以只是上面两步并没什么卵用，我们需要将最终构建出来的包存储下来，这就需要用到travis的发布功能，我们可以选择将其发布到Github Release。

上面的配置中provider:releases就是指定要发布的github上，user和password分别对应Github的账号名及密码（travis需要获取github的读取和写入权限，才能将包发布到github上），file用来指定需要发布的文件，这里也就是我们构建出来的apk，可以发现路劲中添加了$TRAVIS_BUILD_DIR如果不添加会报找不到文件目录的错误，在此$TRAVIS_BUILD_DIR代表构建目录。

skip_cleanup:true这个配置比较重要，travis 在构建完成后默认会清理掉构建生成的文件，而我们需要最终构建出来的apk文件进行发布，所以必须要将其设置为true跳过清理操作。

on用来指定具体的发布时机，在此处tags:true代表在有tag push 时才进行发布。
